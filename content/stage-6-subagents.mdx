## START — 서브에이전트가 뭐냐

**서브에이전트(subagent)** 는 자기만의 시스템 프롬프트·도구·컨텍스트를 가진
별도의 일꾼이다. 메인 대화가 "리뷰 좀 해줘" 같은 일을 만나면, 그 일에
특화된 서브에이전트에게 **위임**한다.

RPG로 치면 파티원이다. 탱커한테 힐을 시키지 않듯, 일에 맞는 클래스를
소환해 맡긴다.

### 왜 쓰나 — 세 가지 이득

- **컨텍스트 격리** — 서브에이전트는 자기 창에서 일한다. 메인 대화가
  잡다한 검색 결과로 더러워지지 않는다. (← [STAGE 3](/stages/stage-3-slash-commands)의 `/clear` 철학과 같다)
- **전문화** — "너는 깐깐한 코드 리뷰어다" 같은 전용 프롬프트로 한 가지를 잘한다
- **도구 제한** — 리뷰어한테는 읽기 권한만. 사고를 원천 차단한다

---

## ROUND 1 — 소환하는 두 가지 방법

### A. `/agents` 명령 (가장 쉬움)

입력창에 `/agents` 를 치면 대화형 마법사가 뜬다. 이름·설명·도구·모델을
골라 만들면 파일이 자동 생성된다. 처음엔 이걸로 시작해라.

### B. 파일로 직접 만들기

서브에이전트는 결국 **마크다운 파일 하나**다. 위치에 따라 범위가 다르다:

- 프로젝트 공유 → `.claude/agents/<이름>.md` (커밋해서 팀과 공유)
- 내 개인용 → `~/.claude/agents/<이름>.md` (모든 프로젝트에서)

## ROUND 2 — 파일 구조 뜯어보기

```markdown
---
name: code-reviewer
description: >
  코드 변경을 리뷰할 때 사용. 버그·엣지케이스·네이밍 일관성을
  본다. PR 올리기 전, 또는 "리뷰해줘" 라고 할 때 호출.
tools: Read, Grep, Glob, Bash   # 생략하면 전부 상속
model: sonnet                   # 생략하면 메인과 동일
---

너는 시니어 코드 리뷰어다. 호출되면:

1. 변경된 파일을 먼저 파악한다 (git diff)
2. 다음을 점검한다 — 버그, 빠진 엣지케이스, 주변 코드와의
   네이밍·패턴 일관성, 불필요한 복잡도
3. 발견한 문제만 `파일:라인` 형태로 짧게 보고한다. 칭찬은 생략.

코드를 직접 고치지 마라. 문제 지적까지가 네 일이다.
```

### 필드 4개만 기억해라

- **`name`** — kebab-case 식별자. 호출할 때 이 이름을 쓴다
- **`description`** — ★가장 중요★. 메인 에이전트는 **이 설명만 보고**
  "이 일을 쟤한테 넘길까" 를 판단한다. **언제 쓰는지**를 구체적으로 적어라
- **`tools`** — 쉼표로 나열한 허용 도구. 생략하면 전부 상속. 좁힐수록 안전
- **`model`** — `opus` / `sonnet` / `haiku`. 리뷰처럼 빡센 건 위로,
  단순 작업은 아래로 ([STAGE 5](/stages/stage-5-models) 참고)

## ROUND 3 — 호출하는 법

### 자동 위임 (기본)

메인 에이전트가 `description` 을 읽고 알아서 넘긴다. 그래서 설명이
모호하면 영영 안 불린다. **"~할 때 사용"** 을 명확히 써라.

### 명시적 호출

직접 지목할 수도 있다:

```text
code-reviewer 에이전트로 지금 변경사항 리뷰해줘.
```

## BONUS STAGE — 잘 만드는 원칙

1. **한 놈당 한 가지 일** — "코드도 짜고 리뷰도 하고 배포도" 하는 만능
   에이전트는 결국 아무것도 잘 못한다. 클래스를 쪼개라
2. **`description` 에 트리거를 박아라** — "리뷰할 때", "테스트 짤 때"처럼
   호출 조건을 동사로. 자동 위임의 성패가 여기서 갈린다
3. **도구는 필요한 것만** — 읽기만 하면 되는 에이전트에 쓰기·실행 권한을
   주지 마라
4. **병렬로 굴려라** — 독립적인 일이면 여러 서브에이전트를 동시에 소환해
   한 번에 끝낸다. 탐색·리뷰처럼 갈라지는 작업에 강력하다

> 메인 에이전트는 결국 **오케스트레이터**가 된다. 자잘한 일은 파티원에게
> 넘기고, 자기는 큰 그림과 판단에 집중한다.

---

## CLEAR! — 이번 판 요약

- 서브에이전트 = 전용 프롬프트·도구·컨텍스트를 가진 **파티원**
- `/agents` 로 만들거나 `.claude/agents/*.md` 파일로 직접
- **`description` 이 자동 위임의 핵심** — 언제 쓰는지를 구체적으로
- 한 놈당 한 가지 일, 도구는 최소로, 독립 작업은 병렬로

이제 진짜 최종 보스 — 팀 전체가 같이 쓰는 법으로 간다. ▶

## START — CLAUDE.md가 뭐냐

세션이 시작될 때 **자동으로 읽히는 프로젝트 설명서**다. 스택, 컨벤션,
하지 말 것을 여기에 적어두면, 매번 프롬프트에 같은 말을 붙일 필요가 없다.

> [STAGE 1](/stages/stage-1-prompting)에서 "제약을 먼저 줘라"고 했다.
> CLAUDE.md는 그 제약을 **영구 저장**하는 곳이다.

---

## ROUND 1 — 어디에 두나

- **프로젝트 루트 `./CLAUDE.md`** — 그 레포 전체에 적용 (팀과 공유, 커밋)
- **`~/.claude/CLAUDE.md`** — 모든 프로젝트에 적용되는 내 개인 취향
- **하위 폴더 `CLAUDE.md`** — 그 디렉터리에서 일할 때 추가로 읽힘

가장 빠른 시작: 프로젝트에서 `/init` 을 치면 초안을 만들어준다.
그다음 손으로 다듬는다.

---

## ROUND 2 — 뭘 적나 (좋은 예)

```markdown
# 프로젝트: Acme 대시보드

## 스택
- Next.js 14 (App Router) / TypeScript / Tailwind
- 상태관리: Zustand (Redux 쓰지 마)
- 테스트: Vitest + Testing Library

## 컨벤션
- 컴포넌트는 함수형 + named export
- 파일명 kebab-case, 컴포넌트명 PascalCase
- API 호출은 src/lib/api/ 안에서만

## 하지 말 것
- 새 의존성 추가 전에 먼저 물어볼 것
- any 타입 금지
- 기존 public API 시그니처 변경 금지

## 명령어
- 개발: pnpm dev
- 테스트: pnpm test
- 린트: pnpm lint
```

## ROUND 3 — 잘 쓰는 원칙

### 1. 짧고 구체적으로

소설 쓰지 마라. **모델이 추측하던 것**만 콕 집어 적는다. 길면 오히려 묻힌다.

### 2. "하지 말 것"이 특히 강력하다

모델은 시키면 한다. **안 했으면 하는 것**을 적어두는 게 사고를 막는다.
`새 라이브러리 추가 금지`, `이 폴더 건드리지 말 것` 같은.

### 3. `#` 로 그때그때 추가

작업 중 "아, 이건 규칙으로 박아두고 싶다" 싶으면 입력창에 `#` 를 치고
적으면 CLAUDE.md에 바로 들어간다.

### 4. 명령어를 적어둬라

빌드·테스트·린트 명령을 적어두면 모델이 **직접 돌려서 검증**한다.
이게 있고 없고가 자동화의 차이를 만든다.

## CLEAR! — 이번 판 요약

- `/init` 으로 시작하고 **짧고 구체적으로** 다듬어라
- **"하지 말 것"** 과 **명령어**가 특히 값지다
- 루트 = 팀 공유, `~/.claude/` = 내 개인 취향
